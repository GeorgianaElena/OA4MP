Dynamic Client Registration

The dynamic client registration specification is supported in OA4MP as of version 4.2. This means that a trusted administrative client can dynamically create regular clients for use with an OA4MP system. If you have not done so already, you should read the section on administrative clients .

How's it work?

We do not support registrations for a simple client to this endpoint. For that you should go to ../oauth2/register. Your request will then be seen by the adminstrator(s) and once approved, you will be notified. This facility is to allow for a site manager who must manage several OAuth clients to do so without having to resort to a vetting process. We get the admin and the the admin can take it from there.

The two specifications that are behind this are RFC 7591 and RFC 7592. The flow is that you register an administrative client. This is initially not approved and any operation you try will fail with a message to that effect. We get an notification and once the admin client is approved, it may be used.
